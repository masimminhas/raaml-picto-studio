RAAML Picto Studio

An open-source Eclipse workbench for multi-technique safety analysis aligned with the OMG Risk Analysis and Assessment Modeling Language (RAAML, OMG formal/2023-06-01). Provides a standalone Ecore metamodel, live Epsilon Picto visualizations, EVL well-formedness validation, a dedicated FMEA form editor, and bidirectional ETL transformations to and from the OMG RAAML UML profile.

What's Included

Standalone Ecore metamodel
138 EClasses, 9 sub-packages covering FMEA, FTA, STPA, HARA, GSN, HAZOP, and the ISO 26262 requirements hierarchy. Carries Ecore annotations mapping each EClass to its OMG RAAML profile stereotype.
Live Picto visualizations
Per-technique HTML and Graphviz views (FMEA risk tables, FTA fault trees, STPA control structures, GSN argument hierarchies, HARA tables, Requirements hierarchy, HAZOP).
Bidirectional ETL round-trips
Pair-1 (library ↔ Ecore) and Pair-2 (user UML ↔ SafetyAnalysis) transformations for all seven techniques. Profile-resolution-independent stereotype detection via AnyType plus base_* generalisation; backward output carries provenance EAnnotations.
EVL validation catalog
15 well-formedness rules derived from AIAG FMEA-4, IEC 61025, STPA Handbook, ISO 26262 Part 3, and GSN Community Standard v3. Batch-runnable via Ant for CI workflows.
FMEA form editor
Eclipse SWT editor with automatic Risk Priority Number computation and live Picto refresh on save.
Autonomous Emergency Braking case study
Reference user models for all seven techniques. Quantitative data grounded in Euro NCAP and ISO 26262 PMHF targets.

Installing via Eclipse

  1. Open Eclipse and go to Help → Install New Software…
  2. Click Add… and enter the following URL:
https://masimminhas.github.io/raaml-picto-studio/
  1. Select RAAML Picto Studio Feature and click Next.
  2. Accept the license and complete the installation.
  3. Restart Eclipse when prompted.

Using the Transformations

The transformations are bundled inside the plugin JAR and can be referenced from your own Ant workflows or Epsilon scripts:

platform:/plugin/raaml-picto-studio/transformations/pair2-usermodel-fmea-forward.etl

Reference Ant runners are provided for each technique (e.g. run-pair2-stpa.xml, run-pair2-hara-backward.xml); adapt the source/target paths to your model files. See the transformations folder on GitHub for the full set.

Requirements

  • Eclipse IDE 2024-03 or later (tested on 2025-06 / 4.36.0)
  • Eclipse Epsilon Picto, ETL, EVL, EGL/EGX ≥ 2.0.0
  • Java 17+
  • Graphviz on PATH (for FTA, STPA, and GSN renderings)
